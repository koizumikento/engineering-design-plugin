# 仕様書テンプレート解説

## 概要

このドキュメントでは、設計仕様書の作成方法とテンプレートの使い方を解説します。

---

## 1. 機械設計仕様書

### 必須セクション

| セクション | 内容 | 注意点 |
|-----------|------|--------|
| 基本情報 | プロジェクト名、バージョン、日付 | ステータス管理が重要 |
| 機能要件 | 何を実現するか | チェックリスト形式 |
| 寸法・形状 | 全長、全幅、全高、公差 | 表形式で明確に |
| 材質・表面処理 | 材質、色、仕上げ | 3Dプリント/切削で異なる |
| 接続・取付 | 穴位置、ねじ規格 | 相手部品との整合性 |
| 出力形式 | STEP/STL/PNG | 用途に応じて選択 |

### 寸法記載のルール

```markdown
| 項目 | 値 | 公差 | 備考 |
|------|-----|------|------|
| 全長 | 100mm | ±0.5mm | 外形寸法 |
| 全幅 | 60mm | ±0.5mm | 外形寸法 |
| 全高 | 40mm | ±0.5mm | 蓋含む |
| 肉厚 | 2.0mm | ±0.2mm | 均一 |
```

### 公差の目安

| 加工方法 | 一般公差 | 精密公差 |
|---------|---------|---------|
| 3Dプリント（FDM） | ±0.5mm | ±0.2mm |
| 3Dプリント（SLA） | ±0.2mm | ±0.1mm |
| CNC切削 | ±0.1mm | ±0.05mm |

## 2. 回路設計仕様書

### 必須セクション

| セクション | 内容 | 注意点 |
|-----------|------|--------|
| 基本情報 | プロジェクト名、バージョン | ステータス管理 |
| 機能要件 | 回路の目的 | 動作条件を明確に |
| 電気的仕様 | 電圧、電流、周波数 | 最小/標準/最大で記載 |
| 部品選定方針 | サイズ、精度、耐圧 | コスト考慮 |
| シミュレーション要件 | DC/AC/過渡解析 | 検証項目 |
| 出力形式 | ネットリスト/SVG/BOM | 後工程を考慮 |

### 電気的仕様の記載例

```markdown
| 項目 | 最小 | 標準 | 最大 | 単位 |
|------|------|------|------|------|
| 入力電圧 | 4.5 | 5.0 | 5.5 | V |
| 出力電圧 | 3.0 | 3.3 | 3.6 | V |
| 出力電流 | - | 100 | 200 | mA |
| 動作温度 | -20 | 25 | 70 | °C |
```

### 部品選定の基準

| 部品 | 選定基準 | 推奨 |
|------|---------|------|
| 抵抗 | サイズ、精度、定格電力 | 1608サイズ、1%、1/10W |
| コンデンサ | 種類、耐圧、温度特性 | 積層セラミック、X7R |
| 半導体 | 汎用/指定、パッケージ | 汎用品優先 |

## 3. 統合設計仕様書

### 基板-筐体インターフェース

統合設計では、基板と筐体の整合性が最も重要です。

```markdown
| 項目 | 基板側 | 筐体側 | 備考 |
|------|--------|--------|------|
| 基板サイズ | 50×30mm | 内寸54×34mm | クリアランス2mm |
| 取付穴位置 | 四隅3mm内側 | ボス位置一致 | M2.5ねじ |
| コネクタ位置 | 短辺中央 | 開口位置一致 | USB-C |
| 最大部品高 | 10mm | 内部高12mm | クリアランス2mm |
```

### 整合性チェックリスト

- [ ] 基板外形 ≤ 筐体内寸 - 4mm（各辺2mmクリアランス）
- [ ] 取付穴位置が完全一致
- [ ] コネクタ開口部の位置・サイズが一致
- [ ] 最大部品高 + 2mm ≤ 筐体内部高さ
- [ ] 放熱経路が確保されている

## 4. 曖昧表現の排除

### NG例とOK例

| NG（曖昧） | OK（具体的） |
|-----------|-------------|
| 適切なサイズ | 100×60×40mm |
| 十分な強度 | 引張強度50MPa以上 |
| 小さい穴 | φ3.2mm貫通穴 |
| 高い精度 | 公差±0.1mm |
| 標準的な抵抗 | 10kΩ ±1% 1608サイズ |

### 検証可能な記述

すべての要件は以下の形式で記載：

```
[対象] は [条件] において [数値] [単位] [以上/以下/±] であること
```

例：
- 筐体の肉厚は全周において2.0mm±0.2mmであること
- 入力電圧5Vにおいて出力電流は100mA以上であること
- 動作温度0〜50°Cにおいて正常動作すること

## 5. 承認フロー

### 仕様書のステータス

| ステータス | 説明 | 次のアクション |
|-----------|------|---------------|
| Draft | 作成中 | レビュー依頼 |
| Review | レビュー中 | 修正または承認 |
| Approved | 承認済み | コード生成可能 |

### 承認チェックボックス

仕様書の最後に必ず記載：

```markdown
## 承認
- [ ] **仕様承認済み** - 承認後に `/engineering-design:cad` または `/engineering-design:circuit` でコード生成を実行
```

このチェックボックスがオンになるまで、コード生成は実行されません。
